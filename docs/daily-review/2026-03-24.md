### PR #85 — `feat: simplify student billing to 2 modes with onboarding wizard`
**Merge:** 03:18 UTC | **Ukuran:** 40 files, +627/-224 | **Scope:** Backend + Frontend

**Apa yang berubah:**
- Billing mode disederhanakan dari 4 mode (manual/per_session/package/subscription) menjadi **2 mode**: `PER_SESSION` dan `MONTHLY_FIXED`
- Admin memilih billing mode saat **onboarding wizard** (locked/irreversible setelah dipilih)
- Tambah cron job `@Cron('0 0 1 * *')` untuk auto-generate tagihan bulanan (MONTHLY_FIXED)
- Hapus `package_fee` dari model Class
- Tambah `billing_period` pada Payment untuk duplicate detection
- Endpoint baru: `GET /api/onboarding/status`, `POST /api/onboarding/billing-mode`, `POST /api/onboarding/complete`
- `POST /api/auth/me` sekarang menyertakan `billing_mode` dan `onboarding_completed`
- SuperAdmin bisa override via `PATCH /api/admin/institutions/:id/billing-mode`
- Migrasi backfill: institusi existing otomatis dapat `PER_SESSION` + `onboarding_completed = true`
- Frontend: ProtectedRoute redirect ke `/onboarding` jika belum completed

**Risiko:** High — perubahan schema DB, logika billing, dan alur admin baru.

## Catatan Risiko

1. **Cron job MONTHLY_FIXED** — belum ada test otomatis. Perlu unit test untuk `InvoiceGeneratorService` dengan mock timer.
2. **Billing mode irreversibility** — jika admin salah pilih, hanya SuperAdmin yang bisa override. Perlu SOP yang jelas untuk tim support.
3. **crypto.randomUUID fallback** (PR #92 mengganti PR #93 sebagian) — ID sekarang pakai `Math.random()` yang tidak collision-proof untuk skala besar. Acceptable untuk form lokal, tapi perlu dicatat untuk masa depan.
4. **`createPortal` dropdown** (PR #86) — perlu regression test di semua halaman yang pakai Select component agar tidak ada side effect dari perubahan DOM structure.
5. **Onboarding wizard vs. PR #93** — PR #93 (merge hari ini 25 Maret, 00:08 UTC) mererestrukturisasi ulang wizard dari 4-step kembali ke 3-step. Test cases di atas (TC-09, TC-10) perlu disesuaikan dengan state terbaru.
