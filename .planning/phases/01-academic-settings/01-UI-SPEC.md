## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`, table cell `px-4 py-3`) |
| lg | 24px | Section padding (`p-6`, modal padding, card padding) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

**Phase-specific spacing notes:**
- Modal padding: 24px (`p-6`) — matches existing Modal component
- Table cell padding: `px-4 py-3` (16px horizontal, 12px vertical) — matches existing AcademicTab room table
- Section dividers inside cards: `pt-8 border-t` or `pt-6 border-t` — matches existing AcademicTab pattern
- Badge pill padding: `px-2 py-1` — matches existing Badge component
- Working day toggle buttons: `px-4 py-2` — matches existing AcademicTab pattern

## Color

All colors use the existing CSS custom properties defined in `src/index.css`. Do NOT use raw Tailwind palette classes (e.g. `zinc-400`) in new code — use semantic tokens.

| Role | Token | Light Value (oklch) | Usage |
|------|-------|---------------------|-------|
| Dominant (60%) | `--background` | `oklch(0.995 0.002 90)` | Page background |
| Card surface | `--card` | `oklch(1 0 0)` | Card backgrounds, modal backgrounds |
| Secondary (30%) | `--secondary` | `oklch(0.965 0.008 90)` | Button backgrounds, table header bg |
| Muted | `--muted` | `oklch(0.960 0.006 90)` | Empty state icon circle bg |
| Primary / Accent (10%) | `--primary` | `oklch(0.555 0.155 180)` (teal) | Primary CTA buttons, ring/focus, active day toggles |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` | Delete room button hover, delete confirmation |
| Success | Badge `success` variant | `bg-emerald-50 text-emerald-600` | Room status "Available" badge |
| Warning | Badge `warning` variant | `bg-amber-50 text-amber-600` | Room status "Maintenance" badge |
| Error | Badge `error` variant | `bg-red-50 text-red-600` | Room status "Unavailable" badge |

**Accent reserved for:** Primary CTA buttons ("Add Room", "Save"), active working day toggle buttons, form focus rings, room edit icon on hover.

**Migration note:** The existing AcademicTab.tsx uses raw `zinc-*` classes (e.g. `text-zinc-400`, `bg-zinc-900`). New and modified code in this phase MUST use semantic tokens (`text-muted-foreground`, `bg-primary`, `text-primary-foreground`) instead. Existing hardcoded zinc references should be migrated to semantic tokens when the component is touched.

## Interaction Contracts

### 1. Working Days Toggles (ACAD-04)

| Property | Specification |
|----------|---------------|
| Element | Row of 7 day buttons (Mon, Tue, Wed, Thu, Fri, Sat, Sun) |
| Active state | `bg-primary text-primary-foreground` (teal filled) |
| Inactive state | `border border-border text-muted-foreground` (outlined) |
| Click behavior | Toggle individual day on/off; minimum 1 day must remain active |
| Persistence | Auto-save via PATCH on each toggle (debounced 500ms), or manual "Save" button at section level |
| Feedback | Sonner toast on save success/failure |
| Default | Mon-Sat active (days 1-6), Sun inactive (day 0) |

### 2. Subject Categories (ACAD-02)

| Property | Specification |
|----------|---------------|
| Display | Horizontal flex-wrap of Badge components (`variant="default"`) |
| Each badge | Category name + X icon (10px) for removal |
| Add action | Plus icon (14px) next to section label opens inline input or small modal |
| Add interaction | Input field with Enter to confirm, Escape to cancel |
| Remove interaction | Click X on badge opens ConfirmDialog (variant "danger") |
| Empty state | "No subject categories yet. Add your first category to organize subjects." |
| Validation | Name required, min 1 char, max 100 chars, no duplicates |
| Persistence | Save full array via PATCH after each add/remove |

### 3. Grade Levels (ACAD-03)

| Property | Specification |
|----------|---------------|
| Display | Horizontal flex-wrap of Badge components (`variant="outline"`) |
| Each badge | Grade name + X icon (10px) for removal |
| Add action | Plus icon (14px) next to section label opens inline input or small modal |
| Add interaction | Input field with Enter to confirm, Escape to cancel |
| Remove interaction | Click X on badge opens ConfirmDialog (variant "danger") |
| Empty state | "No grade levels configured. Add levels to categorize students and classes." |
| Validation | Name required, min 1 char, max 100 chars, no duplicates |
| Default values | SD 1-6, SMP 7-9, SMA 10-12, Universitas (pre-populated on first load) |
| Persistence | Save full array via PATCH after each add/remove |

### 4. Room Management Table (ACAD-01)

| Property | Specification |
|----------|---------------|
| Layout | Full-width table inside a Card, preceded by header row with title + "Add Room" button |
| Columns | Room Name, Type, Capacity, Status, Actions |
| Row hover | `hover:bg-accent/50 transition-colors` |
| Actions column | Edit icon (ExternalLink 14px), Delete icon (Trash2 14px) |
| Edit icon hover | `hover:text-foreground` |
| Delete icon hover | `hover:text-destructive` |
| Status badge | Available = `success`, Maintenance = `warning`, Unavailable = `error` |
| Capacity display | Users icon (14px) + capacity number; "Unlimited" for Online rooms |
| Empty state | EmptyState component with Building2 icon, "No rooms configured", "Add your first room to manage physical and virtual learning spaces.", action = "Add Room" button |

### 5. Room Form Modal (ACAD-01)

| Property | Specification |
|----------|---------------|
| Trigger | "Add Room" button (create) or Edit icon click (edit) |
| Title | "Add Room" or "Edit Room" |
| Max width | `max-w-lg` (existing Modal default) |
| Fields | Name (Input, required), Type (Select, required), Capacity (Input type=number, conditional), Status (Select, required) |
| Type options | Classroom, Laboratory, Studio, Online |
| Capacity field | Hidden when Type = "Online"; shown with placeholder "Max students" for other types; required when shown |
| Status options | Available, Maintenance, Unavailable |
| Footer buttons | Cancel (secondary) + Save (primary, right-aligned) |
| Loading state | Spinner inside Save button, both buttons disabled during save |
| Success feedback | Sonner toast "Room added" or "Room updated", modal closes |
| Error feedback | Sonner toast "Failed to save room. Please try again." |

### 6. Room Delete Confirmation (ACAD-01)

| Property | Specification |
|----------|---------------|
| Component | ConfirmDialog (variant "danger") |
| Title | "Delete Room" |
| Description | "Are you sure you want to delete '{room.name}'? {N} class(es) currently use this room. They will keep the room name until manually updated." — If N=0, omit the class count sentence. |
| Confirm label | "Delete" |
| Cancel label | "Cancel" |
| Loading | Spinner in confirm button during API call |
| Success feedback | Sonner toast "Room deleted" |

### 7. Class Form Room Dropdown (ACAD-05)

| Property | Specification |
|----------|---------------|
| Component | Select (replacing Input) |
| Field label | Room (existing i18n key) |
| Required | Yes (field is required) |
| Options | Rooms with status "Available" only, formatted as "Room Name (Type, N seats)" or "Room Name (Online, unlimited)" |
| Placeholder | "Select a room" |
| Backward compatibility warning | When editing an existing class whose `room` value does not match any settings room: show a yellow warning banner above the Select — `bg-warning/10 border border-warning/20 text-warning-foreground rounded-lg p-3 text-sm` — with text: "This class has a custom room value '{value}'. Please select a room from the list below." |

## Layout Contract

### Academic Tab Structure

The Academic tab is a single scrollable column inside the Settings page tab panel. Internal layout:

```
Card
  +-- Section: Regional & Operational Settings (heading + Calendar icon)
  |
  +-- Working Days (Label + toggle row + helper text)
  |
  +-- border-t divider
  |
  +-- 2-column grid (md breakpoint)
  |     +-- Left: Subject Categories (Label + Plus icon + badge wrap)
  |     +-- Right: Grade Levels (Label + Plus icon + badge wrap)
  |
  +-- border-t divider
  |
  +-- Room Management (subheading + description + "Add Room" button)
  |     +-- Room table (bordered, rounded-xl)
  |           +-- thead (bg-muted)
  |           +-- tbody (divide-y)
  |
  +-- (Room empty state replaces table when rooms array is empty)
```

This layout matches the existing AcademicTab.tsx structure. No structural changes — only wiring to real data and adding interaction handlers.

## Accessibility Notes

- Working day toggles: Use `role="group"` with `aria-label="Working days"`. Each button uses `aria-pressed` to indicate active/inactive state.
- Room table: Standard HTML `<table>` with proper `<thead>`/`<tbody>` — already correct in existing code.
- Modal: Already has `role="dialog"` and `aria-modal="true"`. Escape to close is handled by `useOverlayClose`.
- ConfirmDialog: Already has `role="alertdialog"` and `aria-modal="true"`.
- Form fields: All inputs must have associated Label components with `htmlFor` matching input `id`.
- Delete actions: Must go through ConfirmDialog — no single-click destructive actions.
